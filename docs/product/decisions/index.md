## AY-D-01 V1 项目定位

**决策**：V1 = 长期演进的个人自留地（博客 + 作品集 + 数字花园 + 兴趣表达），**不是**求职 / link-in-bio / 商业品牌。
**可逆性**：低（定位即产品根基）
**来源**：lo-user 17:02 答 "1. b c d e g"（排除 a 招聘 / f link-in-bio）

## AY-D-02 受众优先级

**决策**：自己 > 同行 / 开源协作者 / 读者 > 一般访客；自己舒服 = 一票否决线
**可逆性**：中
**来源**：lo-user 17:02 答 "受众首先是自己看着舒服，然后能让大多数人看着舒服即可"

## AY-D-03 V1 内容范围

**决策**：Must = 自我介绍 / 博客容器 / 联系方式；Nice = now / 简历 / 兴趣展示 / RSS
**可逆性**：高（按需添加）
**来源**：lo-user 17:02 答 "must-have：自我介绍、博客文章、联系方式..."

## AY-D-04 语言

**决策**：中文优先；V1 仅中文，EN 视后续访客需求再加

## AY-D-05 技术栈

**决策**：保留 Nuxt + Vue 生态，V1 升级到最新版（TL 评估）。**atomic CSS 引擎当前实现为 UnoCSS，最终由 #design-system 路径决策（I/II/III）+ Tailwind v4 vs UnoCSS RFC 决定**；III 双轨情况下当下保留 UnoCSS。
**可逆性**：中（迁移成本中等）
**来源**：lo-user 17:02 答 "nuxt 和 vue 生态后续需要先升级到最新" + lo-user 21:47 在 #design-system 触发 Tailwind/UnoCSS 对比 RFC
